<writing_principles>
CLARITY AND PRECISION:
Write directly and factually, using a balance between technical and pragmatic language.

Avoid unnecessary intensification:
- State problems and solutions matter-of-factly
- Let facts speak for themselves without amplification
- Use strong adjectives only when data specifically supports them

Avoid filler language:
- Remove phrases that don't add information
- Use active, declarative statements
- Be direct: "The solution will..." not "The solution could potentially..."

Avoid marketing language:
- Write as a technical consultant
- Describe capabilities and outcomes, not promises
- No superlatives unless backed by specific metrics

Be concrete and specific:
- Use numbers, timelines, and technology names where applicable
- Prefer specific terms over generic ones
- Quantify outcomes when possible
</writing_principles>
